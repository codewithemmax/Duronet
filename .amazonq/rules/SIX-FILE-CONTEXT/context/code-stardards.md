# Code Standards

## General Architecture

- **Decoupled System:** The Next.js frontend (`duronet-client`) and Node.js backend (`duronet-server`) must remain strictly separated. The frontend handles UI and state; the backend handles AI orchestration and external API communication.
- **Fail Gracefully:** If the Gemini API or Fivetran API fails/times out, the UI must display a clear, non-breaking error state rather than crashing the application.

## TypeScript

- **Strict Mode:** Required across both the Next.js and Node.js codebases.
- **Explicit Interfaces:** Define interfaces for all API payloads, AI responses, and GPO data structures in a shared `types/` directory. Avoid `any`.
- **Zod Validation:** Use Zod on the Node.js backend to parse and validate the JSON output from Gemini before sending it to the frontend.

## Next.js Frontend (`duronet-client`)

- **Server vs. Client Components:** Default to Next.js Server Components for layout and static data. Add `"use client"` only for interactive elements (e.g., the Approval Modal, interactive charts).
- **Styling:** Strictly use Tailwind CSS utility classes linked to the variables in `ui-context.md`.
- **State Management:** Keep state as localized as possible. Use React Context or Zustand if sharing the drafted Fivetran payload across multiple sibling components is necessary.

## Node.js Expresss Backend (`duronet-server`)

- **Separation of Concerns:** Keep route definitions (e.g., `routes.ts`) separate from business logic. All Gemini and Fivetran logic must live in dedicated service files (e.g., `services/gemini.ts`, `services/fivetran.ts`).
- **Security:** API keys (`GEMINI_API_KEY`, `FIVETRAN_API_KEY`) must strictly remain in the `.env` file of the Node.js server. They must never be exposed to the Next.js frontend.
- **Streaming:** Utilize streaming for the Gemini API responses to reduce perceived latency on the frontend UI while the agent drafts the pipeline configuration.

## Fivetran Integration (Hackathon MVP)

- **Mocked Execution:** The final POST request to Fivetran should be functionally written in the backend service, but safely stubbed/mocked for the hackathon demo to avoid deploying unverified pipelines to a live data warehouse.
- **Payload Accuracy:** The drafted JSON configuration generated by Gemini must perfectly match Fivetran's official REST API documentation schema for creating a new connector.
