# Architecture Context

## Stack

| Layer            | Technology                                | Role                                           |
| ---------------- | ----------------------------------------- | ---------------------------------------------- |
| Framework (UI)   | Next.js (App Router) + TypeScript         | Frontend application and routing               |
| UI Library       | Tailwind CSS + shadcn/ui                  | Styling and accessible components              |
| Backend          | Node.js + Express/Fastify                 | API orchestration, AI handling, and security   |
| AI Engine        | Gemini 1.5 Pro (Vertex AI SDK)            | Reasoning engine and configuration drafter     |
| Data Integration | Fivetran REST API                         | Target system for drafted pipeline configs     |

## System Boundaries

- `duronet-client/` — Owns the Next.js frontend, UI state, and API client to call the backend. It strictly handles presentation, user interactions, and risk visualizations.
- `duronet-server/` — Owns the Node.js backend. It is strictly responsible for securing API keys, running the Vertex AI/Gemini orchestration, validating Fivetran JSON schemas, and streaming responses to the client.
- `data/` (within server) — Owns the mock static JSON files representing GPO logistics, Manufacturer Reliability Scores, and FDA shortage alerts.

## Storage Model

- **Mock Database (JSON)**: For the hackathon MVP, upstream supply chain alerts and supplier metrics will be stored as structured JSON.
- **Drafted Artifacts**: Fivetran configurations generated by Gemini are temporarily held in server memory or client state until the user approves or dismisses them.
- **NO PHI Storage**: The system absolutely does not connect to or store any hospital Electronic Health Records (EHR) or patient data.

## Auth and Access Model

- **Mock Authentication**: Real authentication is out of scope to save time. The app will hardcode a simulated "Data Engineer / Procurement Admin" session.
- **Access Control**: The UI assumes the active user has administrative rights to review and "deploy" the drafted Fivetran configurations.

## Invariants (Strict Rules)

1. **Decoupled Isolation**: The Next.js client must never contain backend logic or secret API keys. The Node.js server must never render UI components.
2. **Zero-PHI Boundary**: The application and AI models must never ingest, mock, or process patient data. The domain is strictly limited to upstream logistics.
3. **Human-in-the-Loop Enforced**: The system must *never* execute a Fivetran data pipeline automatically. The AI's final output is always a *drafted* configuration payload that requires an explicit user click ("Deploy") to simulate execution.
4. **Latency Handling**: All client-side interactions involving the AI agent must implement clear loading states (e.g., skeletons, streaming text) to account for LLM generation times and prevent timeout frustrations.
